## 已建立的分类体系

### 清酒

**法定分类（特定名称酒）**，用于记录酒款基础信息：

| 类别 | 精米步合 | 特点 |
|------|---------|------|
| 純米大吟醸 | ≤50% | 最高等级，全米酿造 |
| 大吟醸 | ≤50% | 最高等级，含酿造酒精 |
| 純米吟醸 | ≤60% | 全米酿造，吟醸香 |
| 吟醸 | ≤60% | 含酿造酒精 |
| 特別純米 | ≤60%或特殊制法 | 全米，特别标示 |
| 純米 | 无规定 | 全米，无添加酒精 |
| 特別本醸造 | ≤60%或特殊制法 | 含少量酿造酒精 |
| 本醸造 | ≤70% | 含少量酿造酒精 |

**SSI四类型**，用于快速定位风格：

| 类型 | 香气 | 味道 | 典型代表 |
|------|------|------|---------|
| 薫酒 | 高 | 淡 | 大吟醸系、十四代 |
| 爽酒 | 低 | 淡 | 普通酒、本醸造 |
| 醇酒 | 低 | 浓 | 生酛、山廃、燗酒向け |
| 熟酒 | 高 | 浓 | 古酒、熟成系 |

**品评维度**（简化自SAKE DIPLOMA）：
- 外观：清澄度、色调
- 香气：浓度（第一印象）、特征（从固定词汇选取）
- 味道：重量感、甘み、酸み、苦み、バランス、余韵长度
- 结论：SSI类型判断、特定名称推断（如条件允许）

## 记录规范

**清酒**
- 文件位置：`sake/records/都道府県/蔵元名/`，每个蔵元一个文件夹，可另建 `蔵元紹介.md`
- 文件命名：`评分_YYYY-MM_蔵元名_酒款名.md`（评分前缀在最前，日期精确到日时用 `YYYY-MM-DD`）
- 例：`★★★_2025-08_吉村秀雄商店_車坂-山廃.md`

**葡萄酒**
- 文件位置：`wine/records/产国/产区/`，每个产区另建 `产区概览.md`
- 文件命名：`类型-YYYY-酒庄-酒款名.md`（类型：红 / 白 / 橙 / 白浸皮等）
- 例：`红-2025-Ausonia-Montepulciano.md`

**威士忌**（以记录为主，轻品评）
- 文件位置：`whisky/records/产国/产区/酒厂/`，每个产区可建 `产区概览.md`
- 文件命名：`评分_YYYY_酒厂_酒款名.md`（年份可用酒龄或装瓶年；产区概念弱的产国，中间一级可用产国名占位）
- 例：`★★★_2018_Lagavulin_16年.md`

**评分**：★★★ / ★★ / ★ / —（不使用百分制）

**语言**：中文为主，日文专业术语保留原文（如：精米步合、吟醸香）

**香气词汇**：优先使用SAKE DIPLOMA标准词汇，可附加自己的描述

## 给Claude的工作指引

1. **新增酒款记录**时，使用 `sake/templates/sake-tasting.md` 模板（葡萄酒：`wine/templates/wine-tasting.md`；威士忌：`whisky/templates/whisky-tasting.md`；咖啡：`coffee/templates/coffee-tasting.md`）
2. **更新探访场所**时，同步更新本文件的"已探访场所"表格
3. **建议品评词汇**时，优先用SAKE DIPLOMA标准用语，其次用SSI四类型框架
4. **不要**用评分系统以外的方式评价（比如"这款酒很棒"），用具体描述代替
5. 如果用户喝了新品类，先讨论是否建立新的分类体系，再开始记录
